Wayback Archiver

A Python web tool to automatically crawl and archive entire subdomains in the Internet Archive's Wayback Machine.

View on GitHub Try Interactive Demo

Overview

Wayback Archiver helps preserve web content by crawling all pages within a specified subdomain and submitting them to the Internet Archive's Wayback Machine. This is particularly useful for:

  • Preserving content from websites that might be shut down
  • Creating complete historical snapshots of blogs or documentation sites
  • Ensuring important information remains available even if the original site changes
  • Archiving personal projects, portfolios, or academic websites

Features

User-friendly Interface

Easy-to-use web UI for configuring and monitoring archiving jobs

Recursive Crawling

Automatically discovers and follows links within the target subdomain

Smart Filtering

Excludes common paths that would result in duplicate content

Batch Processing

Handles large sites by processing URLs in batches with configurable pauses

Resilient Operation

Retry mechanism with exponential backoff for failed archive attempts

Detailed Logging

Comprehensive logs and output files to track progress

Installation

1. Clone or download the repository:

git clone https://github.com/nickheynen/wayback-archiver.git
cd wayback-archiver

2. Install the required packages:

pip install -r requirements.txt

3. Start the application:

python3 web_interface.py

4. Open your browser and navigate to http://127.0.0.1:5000

Deployment Options

The web interface can be deployed using various hosting options:

PythonAnywhere

  1. Create a free account on PythonAnywhere
  2. Upload the project files or clone the repository
  3. Set up a web app with Flask
  4. Configure your WSGI file to point to the web_interface.py application

Heroku

  1. A Procfile is already included in the repository
  2. Deploy to Heroku using the Heroku CLI or GitHub integration
